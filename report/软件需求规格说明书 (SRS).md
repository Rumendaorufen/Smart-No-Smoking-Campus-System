## 1. 引言 (Introduction)

### 1.1 编写目的

本文档详细描述了“高校无烟校园智能监测系统”的功能需求、非功能需求及接口约束。

* **对于开发人员：** 明确系统架构、算法逻辑及API定义。
* **对于测试人员：** 作为编写测试计划和测试用例的依据。
* **对于项目验收：** 作为最终交付物的验收标准。

### 1.2 项目背景

针对高校宿舍走廊、楼梯间等公共区域吸烟监管难、取证难的问题，利用 YOLOv8-Pose 和 CNN 分类技术，开发一套自动化的智能监测与预警系统。

### 1.3 范围 (Scope)

* **核心功能：** 实时视频流分析、吸烟行为识别、火灾隐患检测、证据视频截取、人工审核后台。
* **监测区域：** 宿舍走廊、消防楼梯间、天台、公共教室（不含宿舍内部及卫生间）。

---

## 2. 总体描述 (Overall Description)

### 2.1 系统架构

系统采用 B/S (Browser/Server) 架构 + 边缘计算节点模式。

* **前端：** Vue.js + TypeScript (管理后台)。
* **后端：** Python Flask (业务逻辑 API)。
* **AI 核心：** Python + PyTorch + OpenCV (推理引擎)。
* **数据库：** MySQL (业务数据) + MinIO/本地文件系统 (视频/图片存储)。

### 2.2 用户角色

| 角色 | 权限描述 |
| --- | --- |
| **保卫处管理员** | 接收实时报警，查看证据视频，进行“确认/误报”审核，导出报表。 |
| **系统运维员** | 配置摄像头 RTSP 地址，调整 AI 阈值，管理系统日志。 |

## 3. 功能需求 (Functional Requirements)

### 3.1 视频采集与预处理模块

| 需求ID | 功能名称 | 详细描述 |
| --- | --- | --- |
| **FR-01** | **RTSP流接入** | 系统必须支持通过 RTSP 协议接入标准监控摄像头视频流（H.264/H.265编码）。 |
| **FR-02** | **多路并发** | 系统应支持单机至少 4-8 路 1080P 视频流的并发解码与分析。 |
| **FR-03** | **环形缓冲区** | 系统需在内存中维护一个时长为 **5秒** 的环形视频缓存队列（FIFO），用于回溯证据。 |

### 3.2 AI 智能分析模块 (核心)

| 需求ID | 功能名称 | 详细描述 |
| --- | --- | --- |
| **FR-04** | **姿态初筛 (Stage 1)** | 使用 **YOLOv8-Pose** 提取人体关键点。<br>

<br>**判定规则：** 当监测到“手腕”关键点与“鼻子/嘴巴”关键点的欧氏距离 < (头部高度 * 0.8)，且动作持续时间 > **1.5秒**。 |
| **FR-05** | **走廊远近过滤** | 系统应支持配置“有效检测距离”或“最小像素阈值”，自动忽略走廊远处过小（如高度 < 50px）的目标，防止误报。 |
| **FR-06** | **局部精细分类 (Stage 2)** | 当 FR-04 触发时，系统自动截取“头部+手部”区域图像 (ROI)，送入 **CNN分类网络**。<br>

<br>**分类结果：** 区分 `Positive`(吸烟) 与 `Negative`(喝水/吃东西/托腮)。 |
| **FR-07** | **火灾隐患检测** | 系统应以低频模式（如每5秒1帧）运行目标检测模型，识别画面中的 **明火** 和 **浓烟**。 |

### 3.3 报警与证据管理模块

| 需求ID | 功能名称 | 详细描述 |
| --- | --- | --- |
| **FR-08** | **证据视频生成** | 当 FR-06 判定为“吸烟”或 FR-07 判定为“火灾”时，系统立即提取缓存的 **前5秒** 视频，并继续录制 **后5秒**，合并生成 **10秒 MP4文件**。 |
| **FR-09** | **抓拍图片存档** | 系统需保存触发报警时刻的原始全景图（画框）和 ROI 局部特写图。 |
| **FR-10** | **报警分级** | **Level 1 (疑似)：** 仅记录日志。<br>

<br>**Level 2 (吸烟)：** 生成视频，标记“待审核”。<br>

<br>**Level 3 (火灾)：** 生成视频，直接触发最高级警报。 |

### 3.4 Web 管理后台模块

| 需求ID | 功能名称 | 详细描述 |
| --- | --- | --- |
| **FR-11** | **实时监控大屏** | 展示所有接入摄像头的实时画面，报警时对应画面红框闪烁提示。 |
| **FR-12** | **报警审核列表** | 展示待审核的报警记录。管理员可点击播放 10秒 证据视频。 |
| **FR-13** | **人工处置交互** | 提供 `[确认违规]` 和 `[误报忽略]` 按钮。<br>

<br>点击确认后，状态变更为“已处置”，并记录审核人姓名。 |
| **FR-14** | **数据可视化** | 统计并展示：今日报警数、高发区域（热力图）、高发时段、处理率。 |

## 4. 非功能需求 (Non-Functional Requirements)

### 4.1 性能要求 (Performance)

* **实时性：** 视频流分析延迟应控制在 **3秒以内**。
* **流畅度：** 在推荐硬件配置下，每路视频的处理帧率 (FPS) 应不低于 **15 FPS**。
* **并发能力：** 单台服务器（配置 RTX 3060）应能稳定支持 **8路** 1080P 视频流分析。

### 4.2 准确性要求 (Accuracy)

* **检出率 (Recall)：** 在光线充足环境下，吸烟行为检出率应 **≥ 90%**。
* **误报率 (False Positive)：** 通过二级分类过滤后，误报率应控制在 **10% 以内**（主要排除吃东西、喝水）。

### 4.3 可靠性 (Reliability)

* **断流重连：** 当摄像头网络中断恢复后，系统应能自动重新连接视频流，无需人工重启服务。
* **存储清理：** 系统应支持配置存储策略（如：仅保留最近 30 天的证据视频），防止磁盘爆满。

### 4.4 隐私与合规 (Constraint)

* **数据隔离：** 所有视频数据仅在校园内网传输与存储，严禁上传公网云端。
* **无死角原则：** 针对宿舍走廊区域，算法对**全画面**进行分析，不设置人为的隐私遮挡（Mask）。

---

## 5. 接口需求 (Interface Requirements)

### 5.1 硬件接口

* **摄像头：** 支持 ONVIF/RTSP 协议的网络摄像机。建议分辨率 1080P，具备红外夜视功能。
* **服务器：** x86 架构 PC 或服务器，需配备 NVIDIA 独立显卡（建议显存 ≥ 6GB）。

### 5.2 软件接口

* **前端交互：** RESTful API (JSON格式)。
* **消息推送：** WebSocket (用于前端实时弹窗)。
* **短信网关：** (预留接口) HTTP POST 接口，用于对接学校短信平台。
