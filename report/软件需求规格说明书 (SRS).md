## 1. 引言

### 1.1 编写目的

本文档定义智慧校园禁烟监控系统的完整功能需求与非功能需求。适用于：

- **开发人员：** 作为实现依据
- **测试人员：** 作为测试用例编写基准
- **项目验收：** 作为交付物验收标准

### 1.2 项目背景

针对高校公共区域吸烟行为监管难、取证难、数据分析难的问题，构建一套集 AI 视觉检测、业务管理、大模型智能问答于一体的综合监控平台。

### 1.3 范围

- **包含：** RTSP 视频流处理、吸烟行为 AI 识别、证据自动录制、报警 WebSocket 推送、人工仲裁审核、历史档案管理、设备与用户管理、系统硬件监控、AI 自然语言数据问答
- **不包含：** 宿舍内部/卫生间等隐私区域监控、手机端 App、第三方短信/钉钉集成

---

## 2. 总体描述

### 2.1 系统架构

四服务协同：

```
Vue 3 前端 (5173)
    │ HTTP / WebSocket
    ▼
Java Spring Boot 中台 (8080)  ←  WebSocket 报警推送
    │                    │
    ├── Python AI 引擎 (5000)     YOLOv8 吸烟检测 + RTSP 流管理
    ├── Python AI Agent (5050)    LangChain SQL 分析助手
    └── MySQL 8.0 (3308)          业务数据 + AI 视图 + 聊天记录
```

### 2.2 用户角色

| 角色 | 权限 |
| --- | --- |
| **管理员 (admin)** | 监控大屏、报警仲裁、档案管理、设备管理、用户管理、系统控制、AI 对话 |
| **操作员 (user)** | 监控大屏、报警仲裁、档案管理（仅可修改自己的审核记录） |
| **未登录用户** | 仅可访问登录页 |

### 2.3 运行环境

| 组件 | 要求 |
| --- | --- |
| 服务器 OS | Windows/Linux（推荐 Linux） |
| Python | 3.9+ |
| Java JDK | 17+ |
| Node.js | 16+ |
| MySQL | 8.0 |
| GPU | NVIDIA 显卡（推荐 6GB+ 显存），CPU 推理亦可降级运行 |
| FFmpeg | 用于证据视频 H.264 转码 |

## 4. 非功能需求

### 4.1 性能

| 指标 | 要求 |
| --- | --- |
| 视频流延迟 | < 500ms（Buffer=1 极低延迟配置） |
| AI 推理延迟 | < 300ms/帧（GPU FP16） |
| WebSocket 报警延迟 | < 2 秒（从检测到前端弹窗） |
| 前端轮询间隔 | 5 秒（设备状态）/ 3 秒（系统状态） |
| 单机并发路数 | ≥ 4 路 1080P（RTX 3060） |
| AI 对话响应 | < 60 秒（含 LLM API 调用） |

### 4.2 可靠性

- RTSP 断流自动重连，重连间隔 2 秒
- 看门狗线程 5 秒超时强制标记离线
- FFmpeg 单线程解码，避免频繁开关流导致的 `pthread_frame.c` 断言崩溃
- 资源释放：先置 `running=False`，0.2 秒缓冲后执行 `cap.release()`
- 数据库连接池：`pool_size=2, max_overflow=1`，防连接耗尽

### 4.3 安全性

- 密码 Bcrypt 加密存储
- JWT 无状态认证，Token 过期需重新登录
- AI Agent 使用专用只读数据库账号（`ai_reader`），仅能访问 4 个视图
- 内部接口与外部接口分离：`/api/internal/**` 走白名单免 Token
- 前端路由守卫 + API 请求拦截器统一注入 Authorization Header
- 跨域 CORS 配置仅允许必要来源

### 4.4 可维护性

- 前后端分离，各服务独立部署、独立启停
- 统一响应格式：`{ code: 200, msg: "success", data: ... }`
- 数据库视图层隔离，AI 查询不直接触碰基表
- Python 模型全局单例，显存占用稳定可预测

### 4.5 存储策略

| 数据类型 | 保留策略 |
| --- | --- |
| 已确认报警 (Status=1) | 永久保留（证据用途） |
| 误报记录 (Status=2) | 保留 30 天（用于模型迭代），之后可清理 |
| 已忽略记录 (Status=9) | 可随时清理 |
| AI 对话记录 | 逻辑删除，`is_deleted=1` |

## 5. 接口需求

### 5.1 外部接口

| 接口 | 协议 | 方向 | 说明 |
| --- | --- | --- | --- |
| RTSP 摄像头 | RTSP/TCP | 入站 | Python 拉流 |
| LLM API | HTTPS | 出站 | web-agent 调用大模型 |

### 5.2 内部服务间接口

| 调用方 | 被调方 | 协议 | 典型接口 |
| --- | --- | --- | --- |
| web-vue | web-back | HTTP REST + WebSocket | `/api/**`, `/ws` |
| web-back | web-agent | HTTP REST | `/api/agent/chat`, `/api/agent/chat/stream` |
| web-flask | web-back | HTTP REST | `/api/internal/alarm/report`, `/api/monitor/devices/sync-status` |
| web-vue | web-flask | HTTP (MJPEG) | `/api/v1/monitor/stream/{id}` |
