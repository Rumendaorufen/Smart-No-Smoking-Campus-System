## 1. 引言 (Introduction)

### 1.1 设计目标

构建一套基于 B/S 架构的智能监控系统。前端负责实时画面展示与报警交互，后端负责视频流处理与业务逻辑，AI 层负责行为识别。系统需支持多路视频流并发、秒级报警推送及完整的证据链回溯。

### 1.2 全栈技术选型

* **前端 (Frontend):** Vue 3 (Composition API) + TypeScript + Vite + Element Plus + Pinia + ECharts。
* **后端 (Backend):** Python 3.9 + Flask + Flask-SocketIO (WebSocket)。
* **AI 核心 (AI Core):** PyTorch + YOLOv8-Pose + OpenCV + Multiprocessing。
* **数据库 (Database):** MySQL 8.0 (业务数据) + Local FS (视频/图片存储)。

### 3.2 核心模块实现细节

#### 3.2.1 视频流与缓存 (`stream_loader.py`)

* **功能：** 维护每个摄像头的 `deque` (双端队列) 缓存。
* **实现逻辑：**
```python
class StreamLoader:
    def __init__(self, rtsp_url):
        self.buffer = collections.deque(maxlen=150) # 5秒缓存
        self.cap = cv2.VideoCapture(rtsp_url)

    def run(self):
        while True:
            ret, frame = self.cap.read()
            if ret:
                self.buffer.append(frame) # 自动挤出旧帧

```



#### 3.2.2 级联检测引擎 (`detector_pose.py` + `detector_cls.py`)

* **功能：** 实现“姿态初筛 + 局部精细分类”。
* **级联逻辑：**
1. **Pose阶段：** 计算 `Wrist` 与 `Nose` 距离。若 `dist < head_size * 0.8` 且持续 `45帧`，触发 Stage 2。
2. **CNN阶段：** 截取 `[y-h:y+2h, x-w:x+w]` (头部扩充区域)，Resize 到 128x128，送入 CNN。
3. **结果：** 若 CNN 输出 `Class 1 (Smoking)` 且置信度 > 0.85，返回报警对象。



#### 3.2.3 证据录制 (`recorder.py`)

* **功能：** 合成 10秒 视频。
* **实现：** 1.  获取报警时刻的 `buffer` (前5秒)。
2.  继续读取 `stream` (后5秒)。
3.  使用 `cv2.VideoWriter` 写入 MP4 文件（编码器推荐 `avc1` 或 `mp4v` 以兼容浏览器）。

### 4.2 核心组件详细设计

#### 4.2.1 实时监控墙 (`Monitor.vue` & `VideoGrid.vue`)

这是系统最复杂的页面，需要处理多路视频流和实时报警特效。

* **HTML 结构：**
使用 `v-for` 根据摄像头列表渲染宫格。
```html
<div class="video-grid">
  <div v-for="cam in cameraList" :key="cam.id" class="video-cell" 
       :class="{ 'is-alarming': alarmingCameras.includes(cam.id) }">
    <img :src="`/api/monitor/stream/${cam.id}`" />
    <div class="cam-name">{{ cam.name }}</div>
  </div>
</div>

```


* **报警闪烁逻辑 (TypeScript)：**
* 在 `onMounted` 中订阅 Pinia 的 WebSocket 状态。
* 当收到 `type: 'SMOKING_DETECTED'` 消息时，将对应的 `camera_id` 加入 `alarmingCameras` 数组。
* CSS 中定义 `.is-alarming` 样式：`animation: blink-red 1s infinite;` (边框红光闪烁)。
* 设置 `setTimeout(() => remove(id), 5000)`，5秒后自动停止闪烁。



#### 4.2.2 报警审核弹窗 (`AuditModal.vue`)

* **功能：** 播放证据视频，提交审核结果。
* **Props 参数：** 接收 `alarmRecord` 对象 (包含 `video_url`, `id`)。
* **快捷键交互：**
```typescript
onMounted(() => {
  window.addEventListener('keydown', handleKeyup);
});

const handleKeyup = (e: KeyboardEvent) => {
  if (e.key === 'Enter') confirmAlert(); // 确认违规
  if (e.key === 'Escape') ignoreAlert(); // 误报忽略
};

```



#### 4.2.3 WebSocket 状态管理 (`store/socket.ts`)

* **功能：** 保持全局长连接，接收服务器推送。
* **实现：**
```typescript
export const useSocketStore = defineStore('socket', {
  state: () => ({ socket: null as Socket | null }),
  actions: {
    init() {
      this.socket = io(import.meta.env.VITE_API_BASE_URL);
      this.socket.on('server_push_alert', (data) => {
        // 触发全局通知 ElNotification
        ElNotification({ title: '发现吸烟行为', message: data.location, type: 'error' });
        // 分发事件给 Monitor 组件
        this.handleMonitorEffect(data.camera_id);
      });
    }
  }
});

```

## 6. 接口设计 (API Interface)

基础路径：`/api/v1`

### 6.1 实时流

* `GET /monitor/stream/<device_id>`
* **响应：** `Multipart/x-mixed-replace` (MJPEG 流)。
* **用途：** 前端 `<img>` 标签直接展示。



### 6.2 报警管理

* `GET /alerts?page=1&status=0`
* **功能：** 获取待审核列表。


* `POST /alerts/<id>/audit`
* **Body:** `{ "action": "confirm" }`
* **功能：** 提交审核结果。



### 6.3 统计数据

* `GET /stats/dashboard`
* **响应:** `{ "today_count": 12, "pending_count": 5, "chart_data": [...] }`
* **用途：** 前端 ECharts 渲染图表。

## 7. 部署与实施建议

1. **开发顺序：**
* **第一步 (AI)：** 跑通 `detector_pose.py`，确保能识别手在嘴边的动作。
* **第二步 (后端)：** 实现 RTSP 拉流和 Flask 接口，把视频流推送到浏览器。
* **第三步 (前端)：** 搭建 Vue 框架，显示视频墙。
* **第四步 (联调)：** 串联报警逻辑，实现弹窗和数据库写入。


2. **性能优化：**
* 前端：使用 `v-show` 而不是 `v-if` 切换视频格，避免频繁销毁重建 DOM 导致视频黑屏。
* 后端：Video Writer 合成视频非常消耗 CPU，务必将其放在独立的线程或进程中执行，不要阻塞 Web 服务。
