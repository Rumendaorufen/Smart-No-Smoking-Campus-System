## 3. web-flask 详细设计（Python AI 视觉引擎）

### 3.1 三线程模型（StreamLoader）

每个摄像头对应一个 `StreamLoader` 实例，包含三个守护线程：

```
StreamLoader(camera_id, rtsp_url)
│
├── _reader_thread        # 拉流线程
│   while self.running:
│       cap.read() → latest_frame
│       信号丢失 → reconnect_requested = True → _connect() 重试
│
├── _processor_thread     # AI 推理线程
│   while self.running:
│       frame = latest_frame.copy()
│       detections = detector.detect(frame)      # 双模型推理
│       detections = _handle_alarm_logic(frame, detections)  # 报警判定
│       frame = _draw_ui(frame, detections)      # 画框
│       recorder.add_frame(frame)                # 写入录像缓冲
│       output_frame = frame
│
└── _watchdog_thread      # 看门狗线程
    while self.running:
        if time.time() - last_read_time > 5s:
            _update_db_status(0) → reconnect_requested = True
```

### 3.2 连接配置（防崩溃关键）

```python
# stream_loader.py L78-86
os.environ["OPENCV_FFMPEG_CAPTURE_OPTIONS"] = \
    "rtsp_transport;tcp|stimeout;3000000|threads;1"
self.cap = cv2.VideoCapture(self.rtsp_url, cv2.CAP_FFMPEG)
self.cap.set(cv2.CAP_PROP_BUFFERSIZE, 1)
```

- `rtsp_transport;tcp`：强制 TCP 传输，避免 UDP 丢包花屏
- `threads;1`：禁用 FFmpeg 内部多线程解码，解决 `pthread_frame.c` 断言崩溃
- `BUFFERSIZE, 1`：极低延迟，牺牲少量吞吐换取实时性

### 6.6 全局 AI 开关双向同步（SystemControl.vue）

```typescript
const handleBeforeAiChange = async () => {
    const target = !systemStats.globalAi
    // 1. 先调 Python 确认 AI 引擎可达
    const res = await aiRequest.post('/api/v1/system/global_ai', { enabled: target })
    if (res.data.code === 200) {
        // 2. 成功后再同步到 Java 数据库
        await javaRequest.post('/system/control/global_ai_db', { enabled: target })
        return true  // 允许 UI 滑动
    }
    // 3. Python 不可达 → 物理拦截滑动
    ElMessage.error('AI 服务器处于下线状态')
    systemStats.globalAi = false
    return false
}
```

```sql
-- ai_alarm_detail_view：报警详情（三表 JOIN）
SELECT a.*, d.name AS device_name, u.username AS auditor_name,
       CASE a.audit_status WHEN 0 THEN '待审核' WHEN 1 THEN '已确认' ... END AS audit_status_text,
       TIMESTAMPDIFF(MINUTE, a.created_at, a.audit_time) AS audit_delay_minutes
FROM alarms a
LEFT JOIN devices d ON d.id = a.camera_id
LEFT JOIN users u ON u.id = a.auditor_id;

-- ai_alarm_daily_stats_view：按日聚合统计
SELECT DATE(a.created_at) AS stat_date,
       COUNT(*) AS total_alarms,
       SUM(CASE WHEN a.audit_status = 0 THEN 1 ELSE 0 END) AS pending_count,
       ...
FROM alarms a GROUP BY DATE(a.created_at);

-- ai_audit_stats_view：审核人效率统计
```
